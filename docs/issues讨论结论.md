# issues 讨论结论

## 1. 依据来源

本文件用于记录 `docs/issues/` 中问题的讨论结果，作为后续代码实现的直接依据。结论来源如下：

1. `docs/issues/issues.txt`
2. `docs/issues/` 下截图
3. `docs/阶段一项目规则与操作流程.md`
4. `docs/review问题与修改计划.md`

## 2. 总体结论

当前问题主要集中在以下几类：

1. 布局与流程图不一致
2. 颜色与地图尺寸不符合预期
3. 提示区、地图框、选中态等视觉细节不符合正式要求
4. 像素字系统尚未实现
5. `Greedy` AI 行为异常，导致得分远低于设计预期

其中：

1. UI 问题应优先按 `docs/MicroCiv流程/` 的图片进行修正。
2. `Greedy` 低分问题应单独作为 AI 逻辑问题处理，不与 UI 改造混在一起。

## 3. 逐条讨论结果

### 3.1 标题 `MicroCiv`

`MicroCiv` 标题应当是巨大的像素字。

1. 标题周围不需要外框。
2. 当前问题是“标题本身缺少像素字表现”。

### 3.2 地图颜色

地图颜色与预期不一致，必须修正。

实现策略固定为：

1. 优先使用 `curses.init_color()` 做精确配色。
2. 若终端不支持 `curses.can_change_color()`，则降级为最接近的近似色继续运行。
3. 不因为颜色精度不足而阻止程序运行。

### 3.3 地图格尺寸

地图格尺寸固定为：

1. 每个地图格使用 `8` 个 `FULL BLOCK`
2. 几何为 `4列 x 2行`

补充约束：

1. 地图格尺寸固定，不做自动缩放。
2. `map_size=24` 若全屏窗口仍放不下，则要求用户放大终端，不通过缩小 tile 或滚动地图解决。

### 3.4 不能修路时的建造面板

当地块不能修路但能建城时：

1. 只显示 `City`
2. 默认选中 `City`
3. 不显示 `Road`

### 3.5 提示信息区域

提示信息必须受右下角提示区约束。

1. 提示文字不能换行溢出到左侧区域。
2. 提示区没有白边框。
3. 提示区位置和尺寸按 `MicroCiv流程/` 图片固定。

### 3.6 选中态

选中地图格时：

1. 只保留闪烁效果
2. 不绘制白色选中边框

### 3.7 `Skip` 位置

`Skip` 的正式位置与样式固定如下：

1. 位于右侧下方
2. 位于提示区上方
3. 使用与其他主要按钮一致的全宽按钮边框样式
4. 按 `MicroCiv流程/` 图片重新对齐

### 3.8 游戏页横向布局基准

游戏主界面的横向布局采用固定右侧按钮列方案。

固定要求如下：

1. 右侧按钮组整体靠近窗口右侧。
2. 按钮列左边界固定，不随地图大小变化。
3. 地图放在左侧可用区域内居中，不反向推动右侧按钮列左右漂移。
4. 这条属于实现稳定性要求，不只是视觉偏好。

采用该方案的原因如下：

1. 布局更稳定。
2. 与像素字“固定字位、固定矩形”的译码器思路一致。
3. 不会因为地图大小变化导致右侧控件水平位置变化。

### 3.9 地图边框

地图显示区域没有白边框。

该要求覆盖：

1. 地图选择页
2. 手动局主界面
3. 自动局主界面
4. 终局页
5. Records 详情页中的最终地图

### 3.10 建造页与研究页遮挡关系

展开建造面板或研究面板时：

1. `Score / Turn` 区域应按流程图被遮挡
2. 不应继续裸露显示在面板旁边

### 3.11 分数异常

当前“单局总分远低于设计预期”的核心问题，不是评分公式整体错误，而是 `Greedy` AI 行为异常。

已核实的现状如下：

1. 在 `map_size=16`、`turn_limit=80` 下，`Random` 大致可达到 `1090 ~ 1629`
2. 同条件下，`Greedy` 仅约 `62 ~ 111`
3. 当前 `Greedy` 常见行为是：
   - 只有 `1` 座城市
   - 大量修路
   - 几乎不造建筑

因此后续处理方向为：

1. 评分公式不作为怀疑对象
2. 先排查 `Greedy` 的策略逻辑与资源转化逻辑

### 3.12 像素字缺失

像素字目前完全缺失，后续必须补做。

### 3.13 Records 旧数据兼容策略

`Records` 不再兼容旧 `baseline` 命名的数据。

固定结论如下：

1. 不再保留 `baseline -> Greedy` 的兼容映射。
2. 不做旧 records 的迁移。
3. 不做旧值的展示层兜底。
4. 当前本地 records 已清空，因此允许直接切断旧兼容链路。
5. 旧文件一律视为不兼容数据。

推荐实现策略固定为：

1. 上调 `RECORDS_SCHEMA_VERSION`。
2. 读取时对 records 关键字段做严格合法值校验。
3. 不兼容文件继续走 `.json.incompatible` 备份并重置为空数据库。

### 3.14 Playback 语义

当前 `Playback = Normal` 与 `Playback = Speed` 没有形成正式的可观看性差异，这需要修正。

固定结论如下：

1. 当前 `Speed` 的表现符合预期，应保持“尽快连续播放”的语义。
2. `Normal` 必须以“可观看”为目标，让观看者能够辨认每一步变化。
3. `Normal` 的正式最小步间隔固定为 `0.20` 秒。
4. 这里的 `0.20` 秒指“本步开始到下一步开始”的总间隔，不是无条件追加的固定 sleep。
5. 若某一步 AI 决策与结算本身已超过 `0.20` 秒，则不再额外等待；优先保证 AI 完整决策。
6. `Normal / Speed` 只影响自动播放节奏，不改变 AI 规则和游戏结果。

## 4. 像素字使用范围

### 4.1 必须使用像素字的内容

以下内容必须使用像素字：

1. 页面标题 `MicroCiv`
2. 主菜单按钮：
   - `Play`
   - `AutoPlay`
   - `Records`
   - `Exit`
3. 地图选择页底部按钮：
   - `Menu`
   - `Recreate`
   - `Start`
4. 游戏内常用动作按钮：
   - `Skip`
   - `Build`
   - `Cancel`
   - `Research`
   - `Restart`
   - `Resume`
   - `Menu`
   - `Exit`
   - `Back`
   - `Delete`
   - `Delete All`
   - `Export`
5. 互斥候选项：
   - `City / Road`
   - `Farm / Lumberyard / Mine / Library`
   - `Agriculture / Logging / Mining / Education`
6. 重点动态数字：
   - `Score` 数值
   - `Turn` 数值
   - 四资源数值
   - 建筑页各建筑数量
   - 其它被定义为重点数字的固定数字位

### 4.2 明确不使用像素字的内容

以下内容不使用像素字，继续使用普通文字：

1. Records 列表页每一条记录的概览信息
2. 普通说明性文本
3. 字段标签与参数标签，例如：
   - `Map Difficulty`
   - `Map Size`
   - `Turn Limit`
   - `Playback`
   - `AI Type`
   - `Score`
   - `Turn`
   - `Food / Wood / Ore / Sci`

## 5. 像素字固定规则

像素字必须遵守以下规则：

1. 采用“固定矩形字位”的译码显示器思路。
2. 每个字的显示区域宽高固定、位置固定。
3. 不允许根据字符不同而调整该字的位置。
4. 重点数字位也使用固定字位。

数字位已明确的要求如下：

1. `Score` 预留 `4` 位
2. `Turn` 预留 `3` 位
3. 资源数量预留 `4` 位
4. 建筑数量预留 `1` 位
5. 其它重点动态数字也应按页面预留固定宽度

这些固定数字位直接约束后续实现中的以下位置：

1. 城市页四资源数值
2. 建筑页中的建筑数量
3. 所有上述数值都必须按固定字位绘制，不能按字符串自然宽度排版

选中态与未选中态规则：

1. 像 `City / Road`、四种建筑、四项科技这类互斥候选项，未选中时仍然使用像素字渲染。
2. 未选中时整组像素字使用 `A_DIM`。
3. 阴影部分也必须一并 `A_DIM`，不能只让主体变暗、阴影保持正常亮度。
4. 选中项显示边框，未选中项不显示边框。

对齐规则：

1. 采用固定字位显示
2. 可使用左对齐或高位补零方案
3. 具体页面应统一，不允许同类数字在不同状态下跳动位置

## 6. 像素字阴影规则

像素字必须带阴影，阴影属于字形设计的一部分。

固定要求如下：

1. 阴影紧贴字体右侧边缘。
2. 如果字体存在封闭区域，则封闭区域内部对应位置也要绘制阴影。
3. 阴影不作为可选装饰，而是像素字正式视觉要求的一部分。

## 8. Records 代码检查结果

结合当前仓库实现，后续为彻底抛弃旧 records 兼容，需要修改的点如下：

### 8.1 `src/microciv/records/models.py`

1. `RECORDS_SCHEMA_VERSION = 2` 需要上调，用于切断旧存档。
2. `RecordDatabase.from_dict()` 当前对 `schema_version` 与 `next_record_id` 使用默认值读取，过于宽松。
3. `RecordEntry.from_dict()` 当前只做类型转换，不校验取值是否合法。
4. 需要新增严格校验：
   - `mode` 只允许 `play / autoplay`
   - `ai_type` 只允许 `Human / Greedy / Random`
   - `playback_mode` 只允许 `"" / normal / speed`
5. 若字段缺失或取值不合法，应直接抛错，让上层按“不兼容文件”处理。

### 8.2 `src/microciv/records/store.py`

1. `load()` 当前的“不兼容文件备份并重置”路径可以继续保留。
2. 后续不再尝试修补或兼容旧文件。
3. 旧 schema、旧 `baseline` 值、缺失关键字段的 records 文件都应统一按不兼容处理。

### 8.3 `src/microciv/curses_app.py`

1. `_record_list_label()` 当前会直接显示 record 中的 `ai_type`。
2. `_record_detail_lines()` 当前会直接显示 autoplay record 的 `ai_type / playback_mode`。
3. 后续不需要再为旧 `baseline` 值做任何显示兼容；这些展示路径只依赖上游已经校验过的正式值。

### 8.4 `tests/test_records.py`

后续测试应补以下场景：

1. 含 `ai_type = "baseline"` 的旧 records 文件会被判定为不兼容，并备份为 `.json.incompatible`。
2. 旧 schema version 的 records 文件会被判定为不兼容。
3. 缺失 `schema_version` 或 `next_record_id` 的 records 文件不再被默认接受。
4. 合法的新 records 文件仍可正常读取、展示、导出。

### 8.5 `src/microciv/curses_app.py` 中的 Playback 现状

1. 当前 `AUTOPLAY_GAME` 只通过每帧推进步数区分 `Playback = Normal / Speed`。
2. 当前 `Normal` 没有最小步间隔控制，因此与 `Speed` 没有形成正式的可观看性差异。
3. 后续实现应新增基于时间的 autoplay 节奏控制，而不是只依赖 `max_steps` 的差异。
4. `Speed` 不增加额外等待；`Normal` 需要满足每步至少 `0.20` 秒的总间隔。

## 9. 后续实现约束

后续代码实现时，必须遵守以下约束：

1. 先修布局、地图框、提示框、`Skip` 位置等 UI 结构问题。
2. 再统一将地图 tile 改为 `4x2`，同步修正 hitbox、地图边框和布局占位。
3. 再实现像素字系统、固定字位和阴影规则。
4. 再处理精确配色与降级方案。
5. 再补齐 `Playback = Normal / Speed` 的正式节奏差异。
6. `Greedy` AI 问题单独处理，不与像素字和布局问题混在一起。

## 10. 验收关注点

后续实现至少应满足以下结果：

1. 标题为无外框的大像素字。
2. 地图格固定为 `4x2`。
3. 无法修路时只显示 `City`。
4. 选中格只闪烁，不出现白色边框。
5. 地图区和提示区都有白边框。
6. 提示文字不会溢出到左侧区域。
7. 建造页和研究页会遮挡 `Score / Turn`。
8. Records 列表概览保持普通文字，不混入像素字。
9. `Skip` 为全宽按钮，且位于提示区上方。
10. 游戏页右侧按钮组左边界固定，不随地图大小变化。
11. 未选中的建筑/科技像素字及其阴影整体使用 `A_DIM`。
12. 幻灯片中的红框/黑框显隐关系与本文件清单一致。
13. 像素字带右侧阴影，封闭区域内部也有对应阴影。
14. `Playback = Speed` 不增加额外人为等待。
15. `Playback = Normal` 在快速决策场景下，相邻两步开始时间间隔不小于 `0.20` 秒。
16. 当 AI 决策本身超过 `0.20` 秒时，`Normal` 不再额外补等待。
17. 含旧 `baseline` 命名或缺失关键字段的 records 文件会被判定为不兼容并备份。
18. `Greedy` 的得分水平应明显高于当前异常状态。
