# 项目正式规则与操作流程

## 1. 文档用途

本文定义当前阶段一项目的正式规则、界面约束、交互方式和操作流程。实现、测试、联调与验收均以本文为直接依据。

## 2. 规则来源与优先级

不同文档负责的内容不同，解释时按以下规则执行：

1. `docs/preparation/fix.txt`
   规定当前阶段任务范围、必须阅读的材料和关键修正规则。
2. `docs/preparation/数值设计.md`
   规定所有正式数值参数，必须严格遵守。
3. `docs/MicroCiv流程/`
   规定所有界面布局、页面切换和操作流程；凡与其它文档存在界面流程冲突时，一律以该目录中的图片为准。
4. `docs/preparation/初始设计.md`
   用于补充项目目标和设计意图；若与后续修正规则冲突，以后续规则为准。
5. `docs/preparation/2026课程设计选题.pdf`
   用于说明课程阶段目标与交付要求。

## 3. 项目目标

项目目标是构建一个基于终端的微型文明模拟器，覆盖以下完整闭环：

1. 地图生成
2. 资源经营
3. 城市扩张
4. 道路与网络连通
5. 建筑建设
6. 科技研究
7. 回合推进
8. 评分结算
9. Records 记录与结果查看
10. 自动经营 AI 对照实验

阶段一当前正式支持的 AI 类型只有：

1. `Greedy`
   规则化/贪心式基线 AI，默认选项。
2. `Random`
   合法动作随机 AI，用于对照。

当前阶段一不在正式交互中开放以下能力：

1. `Custom`
2. 搜索规划 AI
3. 任何以笼统命名替代具体算法说明的 AI 选项

## 4. 地图与邻接规则

### 4.1 地图形态

1. 地图为正方形网格。
2. 不再使用六边形地图。
3. `map_size` 表示边长。
4. 正式候选值固定为 `12 / 16 / 20 / 24`。
5. 默认值为 `16`。
6. 坐标使用 `(x, y)`。

### 4.2 邻接定义

项目中必须区分两类邻接：

1. 八邻域（Moore 邻域）
   城市采集资源使用八邻域，即城市收集周围 `3 x 3` 范围内除自身外的 8 个格子。
2. 四邻域（Von Neumann 邻域）
   道路连通、网络通路、修路合法性与城市直接连通均使用四邻域，只允许共享边连通，不允许对角线连通。

## 5. 地形、占据物与资源

### 5.1 基础地形

1. `plain`
   可建城，可修路，提供 `+2 food / turn`。
2. `forest`
   可建城，可修路，提供 `+2 wood / turn`。
3. `mountain`
   可建城，可修路，提供 `+2 ore / turn`。
4. `river`
   不可建城，可修路，提供 `+1 food / turn` 与 `+1 science / turn`。
5. `wasteland`
   不可建城，可修路，不提供资源。

### 5.2 占据状态

1. `none`
2. `city`
3. `road`

## 6. 地图生成约束

初始地图必须满足以下条件：

1. 初始地图只包含基础地形，不包含城市和道路。
2. `Map Difficulty = Normal` 时，基础地形生成配比固定为：
   - `plain / forest / mountain / river / wasteland = 40 / 22 / 18 / 12 / 8`
3. `Map Difficulty = Hard` 时，基础地形生成配比固定为：
   - `plain / forest / mountain / river / wasteland = 28 / 20 / 24 / 8 / 20`
4. 地图中至少存在 1 个平原格。
5. 荒地不能与河流相邻。
6. 上述配比用于地图生成的基础权重；为满足硬性约束而进行的修正是允许的。
7. 地图应尽量保持基本可玩性和资源分布多样性；该项为设计意图，不作为单独验收条件。

## 7. 城市、道路与网络

### 7.1 城市与道路合法性

1. 新城市只能建在合法空格上。
2. 新道路必须与已有城市或道路四向相邻。
3. 河流不可建城，但可修路。
4. 荒地不可建城，但可修路。

### 7.2 资源采集

1. 城市从自身八邻域中的未被占据格采集资源。
2. 同一个资源格只会按资源类型归属于一个城市计算一次。
3. 多个城市竞争同一资源时，按以下顺序确定归属：
   - 先比较争夺该格的城市自身对应建筑数量：
     - 平原比较 `Farm`
     - 森林比较 `Lumberyard`
     - 山地比较 `Mine`
     - 河流比较 `Library`
   - 对应建筑数量更多者优先
   - 建城更早者优先
   - 坐标字典序更小者优先，即先比较 `x`，再比较 `y`

### 7.3 网络定义

1. 城市本身属于网络节点。
2. 道路属于网络节点。
3. 河流属于网络节点。
4. 网络由城市、道路、河流共同组成。
5. 这些节点都可以位于连通路径上，不区分“端点”和“中间节点”。

### 7.4 网络连通

两座城市属于同一网络，当且仅当满足以下任一条件：

1. 两座城市四向相邻。
2. 两座城市之间存在由城市、道路、河流组成的四向连通通路。

同一网络中的城市共享：

1. 资源池
2. 已研究科技

同一网络中的城市不共享：

1. 已建造建筑数量
2. 单城建筑上限

## 8. 覆盖收益与特殊消耗

### 8.1 覆盖收益

建城或修路覆盖原地形时，一次性获得如下资源：

1. 覆盖平原：`+8 food`
2. 覆盖森林：`+8 wood`
3. 覆盖山地：`+8 ore`
4. 覆盖河流：`+8 food + 8 science`

### 8.2 河流修路消耗

在河流格上修路时，相关网络必须额外支付：

1. `3 wood`
2. 或 `2 ore`
3. 该费用必须从本次落子前即可接入该河流格的既有网络资源池支付。
4. 若该河流格在落子前四向邻接多个可接入网络，则取相邻网络节点中坐标字典序最小者所属网络作为支付网络。

支付顺序固定为：

1. 先尝试木材
2. 木材不足时再尝试矿石

## 9. 断粮、结算与回合

### 9.1 城市消耗

每座城市每回合固定消耗 `4 food`。

### 9.2 断粮规则

若某个网络在本轮结算开始前 `food <= 0`，则该网络进入断粮状态。断粮状态下：

1. 仍可获得食物产出。
2. 其它地形资源停工。
3. 非食物建筑产出停工。
4. 河流格在断粮时只提供 `food`，不提供 `science`。

### 9.3 回合动作

每回合只能执行一个动作。动作集合为：

1. 建城
2. 修路
3. 在指定城市建造建筑
4. 在指定城市所属网络研究科技
5. `Skip`

非法动作必须满足：

1. 状态不变化。
2. 当前回合不推进。
3. 界面给出明确原因。

### 9.4 回合推进顺序

一次成功动作后，系统按以下顺序推进：

1. 执行动作
2. 重算网络关系
3. 重算资源归属
4. 结算地形产出
5. 结算建筑产出
6. 扣除城市粮食消耗
7. 更新分数
8. 判断是否达到回合上限
9. 若未结束，则进入下一回合；若已结束，则直接进入终局页面

### 9.5 回合显示语义

回合显示采用以下统一标准：

1. `turn` 表示当前将要执行的回合。
2. 开局显示第 `1` 回合。
3. 完成第 `N` 回合动作并完成结算后，若 `N` 已达到回合上限，则直接进入终局页面。
4. 终局前不额外显示 `limit + 1`。

## 10. 科技与建筑

### 10.1 科技

1. `Agriculture`
   消耗 `6 science`，解锁农场。
2. `Logging`
   消耗 `12 science`，解锁伐木场。
3. `Mining`
   消耗 `20 science`，解锁矿场。
4. `Education`
   消耗 `20 science`，解锁图书馆。

### 10.2 建筑

每座城市最多建造 `6` 座建筑，同种建筑允许重复建造。建筑参数如下：

1. 农场
   消耗 `5 wood`，提供 `+3 food / turn`
2. 伐木场
   消耗 `5 wood`，提供 `+2 wood / turn`
3. 矿场
   消耗 `6 wood + 4 ore`，提供 `+2 ore / turn`
4. 图书馆
   消耗 `6 wood + 4 ore`，提供 `+2 science / turn`

## 11. 评分规则

总分公式固定为：

`20 * 城市数 + 15 * 建筑数 + 30 * 科技数 + food // 10 + wood // 5 + ore // 5 + science // 20`

评分意图如下：

1. 鼓励稳定扩张城市
2. 鼓励将资源转化为建筑和科技
3. 限制无意义囤积资源的收益

## 12. 界面与视觉约束

### 12.1 布局准则

`docs/MicroCiv流程/` 中的图片定义正式页面结构。实现时必须满足：

1. 所有页面占满全屏窗口。
2. 图中的黑框和红框都用于固定位置与尺寸。
3. 所有控件的位置完全固定，只根据动作来切换。不让控件去适应画面。
4. 黑框表示实际可见边框。
5. 红框表示占位范围，不一定绘制边框。
6. 页面布局按绝对位置理解，不按相对位置理解。
7. 地图在最大尺寸时边长应略小于全屏窗口纵向宽度。
8. 地图尺寸较小时应在标定的红色框范围内居中。在游戏开始前调整，开始后地图位置不变。

### 12.2 文字与图形

1. 黑色文字使用普通文本。
2. 红色文字使用 Unicode block elements 绘制的像素文字，包括字母和数字。
3. 蓝色文字仅为设计说明，不属于最终界面内容。
4. 标题、地图格、重点数字使用块字符实现。
5. 一个地图格为分 2 排的 8 个 FULL BLOCK。

### 12.3 颜色标准

地形与占据物颜色按以下标准执行：

1. 平原：`#D9EC37`
2. 森林：`#76BB40`
3. 山地：`#C2C2C2`
4. 河流：`#52D6FC`
5. 荒地：`#D19D01`
6. 城市：`#FF6250`
7. 道路：`#FFC777`

### 12.4 选中态

1. 选中的地图格采用手动控制的闪烁效果。
2. 不允许使用 `curses.A_BLINK`。

### 12.5 互斥选项表现

所有互斥候选项采用统一表现规则：

1. 所有候选项始终显示。
2. 选中项显示边框，文字正常显示。
3. 未选中项不显示边框，文字使用较暗颜色（A_DIM）。

该规则至少适用于：

1. `City / Road`
2. `Farm / Lumberyard / Mine / Library`
3. `Agriculture / Logging / Mining / Education`

## 13. 输入与快捷键

### 13.1 输入原则

1. 鼠标是主要输入方式。
2. 快捷键是辅助输入方式。
3. 快捷键语义不得与鼠标流程冲突。
4. 所有可见且应可操作的控件都必须真实可点击。

### 13.2 快捷键标准

快捷键统一采用以下标准：

1. `b`
   返回上一层；在 Records 详情页返回列表，在 Records 列表页或空记录页返回主菜单。
2. `m`
   打开游戏内菜单；在游戏内菜单页再次按下时，效果等价于 `Resume`。
3. `d`
   在 Records 列表中跳转到底部。
4. `t`
   在 Records 列表中跳转到顶部。
5. `q`
   仅用于退出程序；退出动作应放在主菜单或明确的退出场景中，不承担层级返回功能。

## 14. 页面流程

### 14.1 主菜单

主菜单对应 `docs/MicroCiv流程/幻灯片1.PNG`：

主菜单上方显示 `MicroCiv`，下方显示：

1. `Play`
2. `Autoplay`
3. `Records`
4. `Exit`

### 14.2 地图选择页

地图选择页对应 `docs/MicroCiv流程/幻灯片2.PNG`。

从 `Play` 或 `Autoplay` 进入地图选择页后：

1. 左侧显示随机生成的地图。
2. 下侧显示 `Menu`、`Recreate`、`Start`。
3. 右侧显示 `Map Difficulty`、`Map Size`、`Turn Limit`。
4. `Menu` 返回主菜单。
5. `Recreate` 重新生成地图。
6. `Start` 进入正式对局。
7. `Map Difficulty` 切换地图难度 `Normal / Hard`，并决定后续生成地图所使用的基础地形配比。
8. `Map Size` 切换地图边长 `12 / 16 / 20 / 24`。
9. `Turn Limit` 切换回合数 `30 / 50 / 80 / 100 / 150`。

若入口为 `Autoplay`，页面中还需配置：

1. `AI Type`
   只允许 `Greedy` 与 `Random`，默认 `Greedy`。
2. `Playback`
   允许 `Normal` 与 `Speed`；两者都执行同一套自动经营规则，仅区别于行动可视化播放速度。
3. `Playback = Speed`
   不增加额外的人为等待时间，以尽快连续播放为目标，不要求观看者能清晰辨认每一步变化。
4. `Playback = Normal`
   以“可观看”为目标，每一步自动行动完成后，到下一步开始前的总间隔至少为 `0.20` 秒。
5. `Normal` 的 `0.20` 秒属于最小步间隔，而不是固定附加等待。
6. 若某一步 AI 决策与结算本身已经超过 `0.20` 秒，则不再额外补等待；优先保证 AI 完整决策。

### 14.3 游戏主界面

`Play` 模式进入 `docs/MicroCiv流程/幻灯片3.PNG` 对应页面：

1. 左侧显示地图。
2. 右侧依次上方显示当前分数与当前回合数。
3. 右侧下方默认显示 `Skip`，并预留提示消息区。
4. 点击地图格后，该格进入选中态。
5. 按 `m` 可进入游戏内菜单。

### 14.4 Autoplay 局内界面

`Autoplay` 模式进入 `docs/MicroCiv流程/幻灯片9.PNG` 对应页面：

1. 左侧显示当前地图，而非终局地图。
2. 右侧上方显示当前分数。
3. 右侧中上方显示当前回合数。
4. 右侧底部提示区显示当前 `AI Type` 与 `Playback`。
5. Autoplay 局内不显示 `Skip`、建造面板或研究面板。
6. 按 `m` 可进入游戏内菜单。

### 14.5 选中非城市格

选中非城市格后的建造页对应 `docs/MicroCiv流程/幻灯片4.PNG`。

选中非城市格后的右侧面板以 `MicroCiv流程/` 图片为准，并执行以下规则：

1. 非法地块保持默认右侧面板，不显示无效建造面板。
2. 若该格可建城和修路，则显示 `City / Road` 选择与 `Build / Cancel`。
3. 若该格只允许其中一种建造，则只显示合法选项。
4. 点击 `Build` 后执行当前选中的建造动作，并结束本回合。
5. 点击 `Cancel` 后撤销当前面板操作，不结束回合。

### 14.6 选中城市格

选中城市格后的城市主面板对应 `docs/MicroCiv流程/幻灯片5.PNG`。

选中城市格后的右侧面板以 `MicroCiv流程/` 图片为准，并执行以下规则：

1. 右侧上方显示该城市所属网络的四种资源。
2. 资源信息按设计稿排列。
3. 右侧下方只显示两个入口按钮：`Buildings` 与 `Technologies`。
4. 点击 `Buildings` 进入建筑子流程。
5. 点击 `Technologies` 进入研究子流程。

### 14.7 建筑子流程

建筑子流程的页面结构、控件位置和切换顺序以 `MicroCiv流程/` 图片为准，并满足：

1. 从城市面板点击 `Buildings` 后，进入 `docs/MicroCiv流程/幻灯片6.PNG` 对应页面。
2. 页面中始终显示 `Farm / Lumberyard / Mine / Library` 四个候选项，以及 `Build / Cancel`。
3. 每个建筑项显示该城市当前对应建筑数量。
4. 四个建筑候选项遵循统一互斥选中态规则。
5. 点击 `Build` 时，必须同时检查单城建筑上限、资源充足性和科技解锁条件。
6. 建造成功则结束本回合并返回默认游戏主界面。
7. 建造失败或点击 `Cancel` 则返回城市主面板，不推进回合。

### 14.8 研究子流程

研究子流程的页面结构、控件位置和切换顺序以 `MicroCiv流程/` 图片为准，并满足：

1. 从城市面板点击 `Technologies` 后，进入 `docs/MicroCiv流程/幻灯片7.PNG` 对应页面。
2. 页面中始终显示 `Agriculture / Logging / Mining / Education` 四个候选项，以及 `Research / Cancel`。
3. 已研究科技使用高亮并加粗（`A_BOLD`, `A_STANDOUT`）显示，且点击无效。
4. 未研究科技遵循统一互斥选中态规则。
5. 点击 `Research` 后，为该城市所属网络研究当前选中的科技，并结束本回合。
6. 研究失败或点击 `Cancel` 后返回城市主面板，不推进回合。

### 14.9 游戏内菜单

游戏内菜单对应 `docs/MicroCiv流程/幻灯片11.PNG`。

游戏内菜单页面以 `MicroCiv流程/` 图片为准，包含：

1. `Menu`
2. `Resume`
3. `Restart`
4. `Exit`
5. 点击 `Resume` 返回进入菜单前的局内页面。
6. 点击 `Menu` 返回主菜单，不保存当前对局进度。
7. 点击 `Restart` 返回地图选择页，并保留当前模式、地图难度、地图尺寸、回合上限，以及 Autoplay 模式下的 `AI Type` 与 `Playback` 配置。
8. 点击 `Exit` 直接退出程序。

### 14.10 终局页

所有回合结束后：

1. `Play` 模式进入 `docs/MicroCiv流程/幻灯片8.PNG` 对应页面。
2. `Autoplay` 模式进入 `docs/MicroCiv流程/幻灯片10.PNG` 对应页面。
3. 左侧显示最终地图。
4. 右侧上方显示最终分数。
5. 右侧提供 `Restart`、`Menu`、`Exit`。
6. `Autoplay` 终局页右侧底部提示区显示 AI 类型与 Playback 类型。
7. 点击 `Restart` 返回地图选择页，并保留当前模式与配置。
8. 点击 `Menu` 返回主菜单。
9. 点击 `Exit` 退出程序。

### 14.11 Records

Records 页面必须支持最多 `10000` 条记录，并以结果浏览为目标。流程如下：

1. 若当前无记录，则进入 `docs/MicroCiv流程/幻灯片13.PNG` 对应的空记录页。
2. 空记录页只显示 `No records` 与 `Back`。
3. 空记录页点击 `Back` 或按 `b` 返回主菜单。
4. 若当前有记录，则进入 `docs/MicroCiv流程/幻灯片12.PNG` 对应的列表页。
5. 列表页按两列展示当前页记录位，空白记录位不绘制边框。
6. 单条列表记录至少显示：`record_id`、日期时间、`Human` 或 AI 类型、最终总分。
7. 列表页支持鼠标滚轮上下滚动。
8. 列表页支持 `d` 跳底、`t` 跳顶。
9. 列表页底部固定显示 `Delete All`、`Export`、`Back`。
10. 点击记录进入 `docs/MicroCiv流程/幻灯片14.PNG` 对应的详情页。
11. 详情页左侧显示该存档的最终地图。
12. 详情页右侧固定显示以下字段：
    - 最终得分
    - 模式
    - 记录时间
    - 地图难度
    - 地图大小
    - 回合上限
    - 实际结束回合
    - 城市数
    - 建筑数
    - 科技数
    - `food / wood / ore / science` 结余
13. 若该记录为 `Autoplay`，详情页还需额外显示：
    - AI 类型
    - Playback
    - 回合总耗时
    - 单步平均耗时
14. 详情页底部固定显示 `Back`、`Menu`、`Delete`。
15. 详情页点击 `Back` 或按 `b` 返回列表页。
16. 详情页点击 `Menu` 返回主菜单。
17. 详情页点击 `Delete` 后立即删除当前记录：
    - 若删除后仍有记录，则返回列表页
    - 若删除后已无记录，则进入空记录页
18. 列表页点击 `Delete All` 后立即清空全部记录，并进入空记录页。
19. 列表页点击 `Export` 后，将当前全部记录导出到 `exports/records_export.json`，覆盖同名旧文件；导出成功后停留在列表页并给出提示。
20. 列表页点击 `Back` 或按 `b` 返回主菜单。

## 15. Records 排序规则

Records 列表排序采用以下固定标准：

1. 默认按最近记录优先显示。
2. 若时间相同，则按 `record_id` 降序。
3. 顶部、底部跳转与滚动都以当前排序后的列表为准。
